Form(폼) 요소

1. Form이란?

Form(폼)은 사용자가 입력한 데이터를 서버로 전송하기 위한 HTML 요소입니다.
로그인, 회원가입, 검색창 등에서 반드시 사용됩니다.
폼 안에 작성된 입력값은 서버로 전달됩니다.

2. Form 기본 문법

<form method="전송방식" action="처리페이지">
   입력 요소들...
</form>
속성설명
method데이터를 서버로 전송하는 방식을 지정합니다.
action데이터를 처리할 서버 또는 HTML 페이지 주소를 지정합니다.

3. method 속성

method는 폼 데이터를 서버로 보내는 방식을 지정합니다. 대표적으로 GET 방식과 POST 방식가 사용됩니다.

방식특징
GET URL 주소 뒤에 데이터가 붙어서 전송됩니다.
예) result.html?name=hong&age=20

물음표(?) 뒤부터 시작되는 name=hong&age=20 부분을 QueryString(쿼리스트링)이라고 합니다.
QueryString은 GET 방식으로 전송된 데이터가 URL에 포함되어 나타나는 영역입니다.

[ QueryString 기호 구조 ]
? : 웹 주소와 데이터(QueryString)를 구분해 주는 기호
= : 변수 이름과 값을 연결하는 기호
& : 여러 개의 데이터를 구분하는 기호

주소창에 데이터가 그대로 보이므로 보안이 필요한 데이터 전송에는 적합하지 않습니다.

검색, 단순 조회 기능에 주로 사용됩니다.
POST 데이터가 HTTP 요청 본문(body)에 담겨 전송됩니다.
주소창에 데이터가 보이지 않습니다.
로그인, 회원가입 등 중요한 데이터 전송에 사용됩니다.

4. action 속성

action은 폼 데이터를 받을 처리 페이지 주소를 지정합니다. 서버 프로그램(PHP, JSP, Spring, Node, Python 등)이 이 데이터를 받아 처리합니다. 

<form method="get" action="result.html">

위 예제는 입력 데이터를 result.html 페이지로 전달합니다.

5. 예제 코드

① GET 방식 예제

<form method="get" action="result.html">
  이름 : <input type="text" name="username">
  <input type="submit" value="전송">
</form>

전송 버튼을 누르면 result.html?username=입력값 형태로 QueryString이 생성됩니다.

② POST 방식 예제

<form method="post" action="result.html">
  아이디 : <input type="text" name="userid">
  비밀번호 : <input type="password" name="userpw">
  <input type="submit" value="로그인">
</form>

입력 데이터가 주소창에 보이지 않고 서버로 전송됩니다.

6. 정리

  • Form은 사용자 입력 데이터를 서버로 전송합니다.
  • method는 데이터 전송 방식을 결정합니다.
  • GET 방식은 QueryString을 URL에 포함하여 전송합니다.
  • QueryString은 물음표(?) 뒤에서 시작됩니다.
  • POST 방식은 데이터가 주소창에 보이지 않습니다.
  • action은 데이터를 처리할 페이지 주소입니다.

자바스크립트 이벤트 · 이벤트 리스너 · 이벤트 핸들러 정리

DOM 요소에서 발생하는 이벤트를 처리하기 위해 “리스너 등록 → 핸들러 실행” 흐름을 이해합니다.

1) 이벤트(Event)란?

이벤트(Event)는 웹 페이지에서 발생하는 “일”입니다. 사용자의 동작이나 브라우저 상태 변화에 의해 발생합니다.

  • 마우스 : click, dblclick, mouseenter, mouseleave
  • 키보드 : keydown, keyup
  • 폼 : input, change, submit
  • 문서/브라우저 : load, scroll, resize

예) 버튼을 클릭하면 “click 이벤트가 발생합니다.”

2) 이벤트 리스너(Event Listener)란?

이벤트 리스너는 DOM 객체에서 이벤트가 발생할 경우 해당 이벤트 처리 핸들러를 추가할 수 있는 오브젝트입니다. 핵심

즉, “이 요소에서 click이 발생하면, 등록된 함수를 실행해 주세요” 라고 등록하는 역할입니다.

아래 문장이 전체 흐름을 잘 요약합니다.

  • 이벤트가 발생하면 등록한 이벤트 처리 핸들러가 실행됩니다.
  • W3C의 표준 방법으로 DOM 객체의 addEventListener() 메소드로 이벤트 핸들러를 등록할 수 있습니다.
3) 이벤트 핸들러(Handler)란?

이벤트 핸들러는 이벤트가 발생했을 때 실행되는 함수입니다. (처리 함수, 이벤트 처리 함수라고도 합니다.)

예) click 이벤트가 발생했을 때 “알림창을 띄우기”, “색상 변경”, “데이터 전송” 등을 수행하는 함수가 핸들러입니다. 

// 이벤트 핸들러(함수) 예시
function changeflowerf(e){
  alert("click 이벤트가 발생했습니다.");
}
4) addEventListener(type, function_name, useCapture) 형식

형식) addEventListener(type, function_name, useCapture)

  • (1) type : 이벤트 타입을 나타내는 문자열로 'click', 'load', 'keydown' 등등
  • (2) function_name : 이벤트가 발생할 때 실행할 함수명
  • (3) useCapture : true 이면 이벤트 흐름 중 캡처 단계에서 실행될 이벤트 핸들러를 등록합니다.
    false 이면 버블 단계에서 실행될 이벤트 핸들러를 등록합니다. 생략 가능하며 디폴트는 false입니다.

예) addEventListener('click', changeflowerf) 

// 예시: 버튼 클릭(click) 이벤트에 핸들러(changeflowerf) 등록
const btn = document.querySelector("#btn");
btn.addEventListener("click", changeflowerf); // useCapture 생략 → 기본 false(버블 단계)
5) 정리: 이벤트 처리 흐름
  1. DOM 요소에서 이벤트가 발생합니다. (예: 버튼 클릭)
  2. 해당 요소에 등록된 이벤트 리스너가 이벤트를 감지합니다.
  3. 등록된 이벤트 핸들러(함수)가 실행됩니다.

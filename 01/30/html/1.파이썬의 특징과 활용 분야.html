Python의 특징

1) 인터프리터 기반 하이브리드 실행 구조

바이트코드 PVM __pycache__ .pyc
핵심 개념 Python은 실행 시점에는 “인터프리터처럼” 동작하지만, 내부적으로는 소스 코드를 바이트코드(bytecode)로 만든 뒤 Python Virtual Machine(PVM)이 이를 해석(인터프리팅)하여 실행합니다. 즉, 컴파일(중간 코드 생성) + 인터프리팅이 결합된 구조입니다.

① 실행 흐름(큰 그림)


소스 코드 (.py)
  ↓ (1) 컴파일: 바이트코드 생성
바이트코드 (.pyc, 또는 메모리 상 바이트코드)
  ↓ (2) PVM이 바이트코드를 해석(인터프리팅)
프로그램 실행

② 단계별로 더 자세히 보기

  1. 1
    소스 코드 로딩
    Python 실행기가 .py 파일을 읽어 들입니다.
  2. 2
    바이트코드 컴파일
    실행을 위해 소스 코드를 바이트코드로 변환합니다. (이 바이트코드는 CPU 기계어가 아니라 “가상 머신용 중간 코드”입니다.)
  3. 3
    PVM에서 인터프리팅
    PVM이 바이트코드를 한 단계씩 해석하여 실행합니다.
  4. 4
    (필요 시) 바이트코드 캐시
    재사용될 가능성이 높은 경우 바이트코드를 디스크에 저장해 다음 실행을 빠르게 합니다.

③ .pyc 파일은 언제 생성될까?

주로 생성되는 경우: import 되는 “모듈”
  • import 로 불러오는 파일은 재사용될 가능성이 높습니다.
  • 그래서 Python은 해당 모듈의 바이트코드를 __pycache__/에 .pyc로 저장합니다.
  • 다음 실행부터는 소스 재컴파일을 줄이고 실행을 시작할 수 있습니다.
main.py
mymodule.py

python main.py 실행
→ __pycache__/mymodule.cpython-311.pyc 생성
기본적으로 생성되지 않는 경우: 직접 실행한 “메인 스크립트”
  • python main.py 처럼 직접 실행한 파일은
  • 바이트코드로 변환되어 실행되지만, 기본 설정에서는 .pyc로 디스크에 저장하지 않는 경우가 많습니다.
  • 즉, 바이트코드는 메모리에서 만들어 실행되고 종료 시 사라질 수 있습니다.

④ 하이브리드 방식의 장점/의미

  • 편리함: 수정 후 바로 실행됩니다(인터프리터 스타일로 동작합니다).
  • 효율: 중간 코드(바이트코드)를 사용하여 반복 실행에서 부담이 줄어듭니다.
  • 플랫폼 독립성: OS/CPU가 달라도 PVM 위에서 실행될 수 있습니다.

⑤ 구현체(파이썬 코드를 실행해 주는 프로그램)

핵심 개념 파이썬은 언어(문법)는 같지만, 실제로 코드를 실행하는 “구현체(엔진)”는 여러 종류가 있습니다. 환경/목적에 따라 적절한 구현체를 선택할 수 있습니다.
구현체 기반/실행 환경 특징
CPython C 기반 가장 널리 쓰이는 표준 구현체입니다. 일반적으로 “파이썬”이라 부르면 CPython을 의미합니다.
PyPy JIT(Just-In-Time) 기반 JIT 최적화 등을 통해 성능 개선을 시도하는 구현체입니다. 반복 계산이 많은 코드에서 빠를 수 있습니다.
Jython JVM(Java) 파이썬 코드를 JVM 위에서 실행합니다. Java 라이브러리를 활용할 수 있습니다.
IronPython .NET(CLR) .NET 환경에서 파이썬을 실행합니다. C# 등 .NET 라이브러리와 연동할 수 있습니다.
MicroPython 임베디드/마이크로컨트롤러 아주 작은 장치(IoT 등)에서 동작하도록 가볍게 만든 구현체입니다. 표준 라이브러리가 축소되어 있을 수 있습니다.
Stackless Python CPython 변형 코루틴/동시성 처리 등 “스택 없는 실행 모델”에 초점을 둔 구현체입니다.
정리
대부분의 입문/일반 개발에서는 CPython을 사용합니다.
성능 튜닝이 필요하면 PyPy를 고려할 수 있고,
특정 실행 환경(JVM/.NET/임베디드)에서는 해당 구현체(Jython/IronPython/MicroPython)를 사용합니다.

2) 동적 타이핑(Dynamic Typing)

정의 Python은 동적 타이핑(Dynamic Typing) 언어입니다.
변수에 “자료형을 고정해서 선언”하지 않고, 값이 대입되는 순간 그 값의 타입이 결정됩니다.

예시(값이 바뀌면 타입도 바뀝니다)

x = 10
print(type(x))   # <class 'int'>

x = "hello"
print(type(x))   # <class 'str'>

장점

  • 타입 선언 없이 간결한 코드 작성
  • 빠른 개발과 실험 가능

주의점

  • 타입 오류는 실행 중 발생
Python의 모든 값은 객체(Object)입니다. 기본 자료형도 내부적으로는 객체입니다.

3) 간결한 문법 (들여쓰기)

중괄호({}) 대신 들여쓰기로 코드 블록을 구분합니다.

if True:
    print("Hello")
  • 코드 구조가 눈에 잘 들어옵니다.
  • 들여쓰기가 문법이므로 공백/탭 혼용을 피하는 것이 좋습니다.

4) 객체지향 & 함수형 프로그래밍 지원

클래스 기반 객체지향 + 함수형 스타일 모두 지원합니다.

  • 클래스/객체로 구조화 가능합니다.
  • 함수, 람다 등을 활용할 수 있습니다.

5) 풍부한 라이브러리

웹, 데이터 분석, 인공지능, 자동화 등 다양한 분야에서 활용 가능한 라이브러리가 풍부합니다.

  • 표준 라이브러리로 기본 기능만으로도 많은 작업이 가능합니다.
  • 서드파티 패키지(표준 라이브러리가 아닌 외부 라이브러리 - 예) pip install requests)로 분야별 생태계가 매우 큽니다.

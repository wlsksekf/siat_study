JavaScript try...catch 예외 처리 교안

프로그램이 “에러로 멈추는 상황”을 잡아서, 사용자에게 안내하고 정상 흐름을 이어가도록 만드는 문법입니다.

1. 예외(Exception)란?

자바스크립트를 실행하는 도중 정상적인 흐름을 더 이상 진행할 수 없는 상황이 발생할 수 있습니다. 이때 발생하는 비정상 상황을 예외(Exception)라고 합니다.

예외가 처리되지 않으면 프로그램은 즉시 중단됩니다. 따라서 예외를 안전하게 처리하기 위해 try...catch 문법을 사용합니다.

2. 오류(Error)와 예외(Exception)의 구분

자바스크립트에서는 흔히 오류(Error)와 예외(Exception)를 비슷하게 부르지만, 개념적으로는 다음과 같이 구분하여 이해하면 좋습니다.

구분 설명 예
오류(Error) 프로그램 실행 중 발생한 문제 상황 전체를 의미합니다. ReferenceError, TypeError 등
예외(Exception) 오류 중에서 try...catch로 잡아서 처리할 수 있는 오류를 의미합니다. try...catch로 처리되는 Error 객체
정리하면,
모든 예외는 오류이지만, 모든 오류가 예외 처리되는 것은 아닙니다.

3. try...catch의 역할

try...catch는 예외(Exception)가 발생했을 때 프로그램이 중단되지 않도록 처리하는 문법입니다. 

try {
  // 예외가 발생할 수 있는 코드
} catch (err) {
  // 예외가 발생했을 때 실행되는 코드
}

자바스크립트에서 예외가 발생하면 자바스크립트에서 자동으로 Error 객체를 생성하고 그 객체가 catch 블록의 err매개변수로 전달됩니다.

4. 에러 객체(err)에서 자주 보는 값

속성 의미 예
err.name 에러 종류 이름 "TypeError", "ReferenceError" 등
err.message 에러 설명 메시지 "Cannot read properties of undefined" 등
err.stack 에러가 발생한 호출 경로(디버깅용) 브라우저 개발자도구에서 확인
try...catch는 “모든 문제를 해결”하는 문법이 아니라, “터지는 상황을 잡고 안내/복구”하는 문법입니다.

5. throw로 “내가 직접” 에러 만들기

사용자가 잘못 입력했을 때, 조건을 검사하고 의도적으로 에러를 발생시켜 catch로 흐름을 넘길 수 있습니다. 

function parseAge(input) {
  const n = Number(input);

  // 숫자가 아니거나 0 미만이면 "직접" 예외를 던집니다.
  if (Number.isNaN(n) || n < 0) {
    throw new Error("나이는 0 이상의 숫자여야 합니다.");
  }

  return n;
}

try {
  const age = parseAge("abc"); // 여기서 throw 발생
  console.log("나이:", age);
} catch (err) {
  console.log("에러:", err.message);
}

6. 정리

  • try : 에러가 날 수 있는 코드를 실행합니다.
  • catch(err) : 에러가 발생했을 때 실행됩니다. err.message 등으로 안내합니다.
  • finally : 성공/실패와 관계없이 항상 실행됩니다(정리 작업에 사용됩니다).
  • throw : 조건이 맞지 않을 때, 직접 예외를 발생시켜 catch로 흐름을 넘깁니다.

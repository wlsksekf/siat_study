자바스크립트(JavaScript) 역사 및 표준

언어의 탄생 → 표준화 → 현대 웹 개발까지의 흐름

1. 자바스크립트란?

자바스크립트는 웹 브라우저에서 동작하는 프로그래밍 언어입니다.
HTML이 구조를 담당하고, CSS가 디자인을 담당한다면, 자바스크립트는 동적인 동작과 로직을 담당합니다.

현재 자바스크립트는 웹 브라우저, 서버(Node.js), 모바일 앱, 데스크탑 앱에서 모두 사용됩니다.

2. 자바스크립트 탄생 배경

1995년 넷스케이프(Netscape)에서 브렌든 아이크(Brendan Eich)가 단 10일 만에 자바스크립트를 개발합니다.
초기 이름 Mocha → LiveScript → JavaScript (당시 Java 인기에 편승한 마케팅 목적)

※ 자바(Java)와 자바스크립트는 이름만 비슷하고 완전히 다른 언어입니다.

3. 브라우저 전쟁과 표준의 필요성

  • 1990년대 후반 → 넷스케이프 vs 마이크로소프트(IE) 경쟁
  • 브라우저마다 자바스크립트 구현이 달라 혼란 발생
  • 표준화 필요성이 대두됩니다.
💡 이 문제를 해결하기 위해 ECMAScript 표준이 만들어집니다.

4. ECMAScript 표준이란?

1. ECMAScript는 European Computer Manufacturers Association Script의 약자입니다.
즉, 유럽 컴퓨터 제조협회(ECMA)가 정한 스크립트 언어 표준이라는 의미입니다.

2. ECMAScript는 자바스크립트의 공식 표준 규격입니다.
이 표준은 ECMA 국제 표준 기구에서 관리합니다.

✔ JavaScript는 ECMAScript 표준을 구현한 언어입니다.
✔ 표준이 정해지면 모든 웹 브라우저가 이를 따라 구현합니다.

5. ECMAScript 주요 버전 연대표

버전 연도 주요 특징
ES1 1997 최초 표준 제정
ES3 1999 정규식, try-catch 추가
ES5 2009 JSON, strict mode 도입
ES6 (ES2015) 2015 let/const, 화살표함수, class, module 등 대혁신
ES2016~ 매년 async/await, spread, optional chaining 등 지속 추가

6. ES6(2015)가 중요한 이유

  • 현대 자바스크립트 문법의 기준점이 됩니다.
  • React, Vue, Node.js 등 최신 프레임워크 기반 문법입니다.
  • 현재 실무 개발의 사실상 표준입니다.
💡 따라서 "ES6 문법"을 배우는 것은 현대 자바스크립트를 배우는 것과 같습니다.

7. 자바스크립트 실행 환경의 확장

1995 브라우저에서만 실행
2009 Node.js 등장 → 서버에서도 실행
현재 웹, 서버, 앱(React), IoT까지 확장

8. 정리

  • 자바스크립트는 1995년에 탄생합니다.
  • 브라우저마다 구현 차이로 표준이 필요했습니다.
  • ECMAScript가 자바스크립트의 공식 표준입니다.
  • ES6(2015) 이후 현대 개발 문법이 완성됩니다.
  • 현재 자바스크립트는 모든 플랫폼에서 사용됩니다.
🎯 한 줄 요약 JavaScript = ECMAScript 표준을 따르는 현대 웹 개발의 핵심 언어입니다.
